# BookClaw Bug Audit — 2026-06-25

Whole-codebase bug hunt (6 parallel subagent finders over `gateway/`, `frontend/`, `mcp/`). Ranked most-critical first. Status legend: **TO-FIX** (valid, will fix) · **FIXED** · **SKIP** (not a real bug / accepted under the single-user-LAN threat model, with rationale).

## Critical

### C1 — MCP `create_project` always 400s
`mcp/src/tools/projects.ts:17-27` POSTs `{task, template}` but `/api/projects/create` reads `{type, title, description}` (`projects.routes.ts:92-95`) → every call returns 400. The headline "create a project" MCP tool is non-functional. **TO-FIX** (map task→title/description in lockstep).

### C2 — MCP `create_pipeline` always 400s
`mcp/src/tools/projects.ts:29-39` POSTs `{idea, pipeline}` but `/api/pipeline/create` reads `{title, description, personaId, config}` (`projects.routes.ts:218-221`) → every call 400s; the full book pipeline can't start via MCP. **TO-FIX**.

## High

### H1 — Consistency canon leaks across worldless books
`fact-store.ts:76` `priorFacts` canon branch is `source='canon' AND world IS NULL` with **no `book_slug` constraint**; the single shared `consistency.db` then pulls Book B's bible facts when auditing worldless Book A → false high-severity `canon-divergence` findings citing the wrong book. Sibling queries (`reverseIndex`, `orphanCanonFacts`) scope correctly via `OR book_slug = ?`. **TO-FIX**.

### H2 — `skipStep` not parallel-group aware
`projects.ts:1062-1088` activates only the first `pending` step and, finding none, marks the project `completed` without checking still-`active` siblings → skipping one member of a `{parallel:[...]}` group abandons in-flight siblings and ends the project early. `completeStep` already uses `runnableSteps()`; `skipStep` was left on old logic. **TO-FIX**.

### H3 — Unguarded async route handlers (reliability class)
Many `async` handlers `await` IO with neither try/catch nor an `asyncHandler` wrapper; on rejection the Express error middleware never fires (it only runs on `next(err)`) and the `unhandledRejection` handler only logs → **the HTTP request hangs forever and the `res` leaks**. Confirmed sites:
- `books.routes.ts:172-183,186-194,253-262,265-271` (file write/versions; `asyncHandler` not even imported)
- `knowledge.routes.ts:153,159,254,502,541,548,81,110,758,774,784,806,883,890,898`
- `export.routes.ts:105,122` (shell-outs to Python — most failure-prone)
- `worlds.routes.ts:89-111,114-137`
- `personas.routes.ts:95-101`
- `series.routes.ts:30-115,181-187`
- `media.routes.ts:296-327,331-358,389-405`
- `settings.routes.ts:72-139` (the `reinitialize()`), `core.routes.ts:135-142` (`resetLifetime`)
**TO-FIX** (wrap each in try/catch → 500, matching the guarded siblings in the same files).

### H4 — `Series.tsx` cross-series corruption via stale fetch
`Series.tsx:55-59` world-building fetch on `selId` change has no stale-response guard → a slow earlier fetch resolves after a fast later one, loading series A's data while series B is selected; saving then PUTs A's content to B's id. **TO-FIX** (cancelled-flag guard).

### H5 — `PromptRunner.tsx` cross-book write-back
`PromptRunner.tsx` write-back (`confirmReplace`/`saveAsNew`) uses live `slug`/`file` while the Book/File selectors stay enabled during a multi-second run → output from book A can be written into book B. **TO-FIX** (use the captured run context / disable selectors while running).

## Medium

### M1 — `promptCache` unbounded memory leak
`router.ts:164,434` — the `provider:sha256` cache only ever grows (stored `timestamp` never read for eviction). **TO-FIX** (cap size / TTL evict).

### M2 — Gemini/Claude completions don't check `response.ok` before `.json()`
`router.ts:614,689` — a non-JSON error body (502 HTML, empty 503 after retries) throws an opaque `SyntaxError` instead of a descriptive provider error; the OpenAI-compatible path guards correctly (`:785`). **TO-FIX** (mirror the `!response.ok` text() guard).

### M3 — Sandbox `validatePath` doesn't resolve symlinks
`sandbox.ts:25-54` uses string `path.resolve` only (no `realpath`), yet the comment claims it catches symlinks → a symlink under `workspace/` pointing outside passes. No current symlink-creation path, but defense-in-depth + misleading comment. **TO-FIX** (realpath check + correct comment).

### M4 — `extractFromOpening` merge mode drops curated promises
`plot-promises.ts:297` — with `merge:true` (default) it replaces the whole list, keeping only promises whose title the LLM re-extracts verbatim → author-added/confirmed promises are lost on re-run, contrary to the doc comment. **TO-FIX** (append/union, dedupe by lowercased title).

### M5 — Bridge word-count continuation drops the step's model override
`index.ts:2270-2273` — continuation passes `undefined` provider/model, so a pinned cheap-draft/premium-edit model is used only for the first pass → voice seam mid-chapter. The studio routes path passes it correctly. **TO-FIX**.

### M6 — Post-restart context uses truncated step results
`projects.ts` persists `step.result` capped at 500 chars (`:184-187`) and never re-hydrates the full output from disk → after a restart, polish/revision context (which injects `writeStep.result`) gets a 500-char fragment. **TO-FIX** (re-read full output from the data dir when the truncation marker is present).

### M7 — `allocateSlug` TOCTOU
`book.ts:497-504` returns a candidate slug without atomically claiming the dir (unlike `claimSlug`) → concurrent creators can collide and clobber. **TO-FIX** (atomic mkdir claim).

### M8 — Plot-promises audit ignores explicit `?progress=`
`knowledge.routes.ts:579` — `project?.progress ?? (finite q ? q : 100)` means a stored `progress` (incl. `0`) always wins over the query param. **TO-FIX** (honor the explicit param first).

### M9 — `heartbeat.routes.ts` autonomous config unvalidated
`:44-50` forwards `intervalMinutes`/`maxStepsPerWake`/quiet hours from `req.body` with no numeric coercion/range check → `"abc"`/`0`/negative wedges the scheduler. **TO-FIX** (coerce + range-validate → 400).

### M10 — `Files.tsx` preview stale fetch + tab-agnostic refresh
`Files.tsx:71-82` preview fetch has no stale guard (wrong body under a header); `:134-142` `del()` always `loadDocs()` regardless of tab. **TO-FIX** (stale guard + tab-aware refresh).

### M11 — Telegram pause/stop is global, not per-chat
`telegram.ts:45,670,691` + `index.ts:2454-2485` — `pauseRequested` is one bridge-wide boolean and `/stop` picks the first global `active` project → with multiple chats, one chat's `/stop` can pause another's. Lower urgency under single-user use. **TO-FIX** (key by channel; scope to the issuing chat's book).

## Post-fix code review (high-effort workflow)

A full review of the fix batch found **3 CONFIRMED regressions introduced by the fixes** (all fixed) + 3 PLAUSIBLE cleanups (accepted, low) + 18 refuted:

- **R1 (telegram per-chat /stop fallback)** — the M11 fix's "fall back to all projects when the chat owns none" re-enabled cross-chat pausing after a restart. Fixed: the fallback now considers only **unclaimed** projects, and `/stop`/continue act only on an **unambiguous** (exactly-one) unclaimed match — never another chat's run. **FIXED**.
- **R2 (book-transfer orphan slug)** — the M7 fix made `allocateSlug` side-effecting (atomic dir claim); a failed `finalizeImport` between claim and move would orphan the empty dir and consume the slug. Fixed: `finalizeImport` now rolls back the claimed dir on any failure before the staged book is moved in. **FIXED**.
- **R3 (worlds doc delete idempotency)** — the L15 fix returned 404 for a missing document, but the studio's `api()` throws on non-2xx, breaking delete-from-a-stale-list. Fixed: keep the new **world-existence 404**, but a missing document within an existing world returns idempotent `200 {deleted:false}`. **FIXED**.

Accepted PLAUSIBLE cleanups (low, not regressions): skipStep/completeStep share near-identical frontier logic (both test-guarded; extraction deferred to avoid destabilizing the just-fixed scheduler); `validatePath` now does synchronous `realpath`/`existsSync` (required by the symlink-escape fix; sole caller is the chat read/save path, not a hot loop); `StructureLength` duplicates the review-fetch between `load()` and the guarded effect (functional stale-guard is in place).

Post-regression-fix verification: `tsc` clean, full unit suite **1058/1058** (stable across repeated runs; one earlier single failure was an environmental teardown flake unrelated to these edits), `npm run build:frontend` green, MCP build + smoke pass.
